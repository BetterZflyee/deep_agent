# 贡献指南

欢迎你为 deep_agent 做出贡献！本仓库采用 Fork + Pull Request 工作流。

## 提交流程
1. Fork：在仓库主页点击 `Fork`，复制到你的账户
2. 创建分支：在你的 Fork 中创建分支，如 `feature/xxx` 或 `fix/yyy`
3. 提交改动：尽量小步提交，写清楚提交信息（建议参考 Conventional Commits）
4. 推送分支：将分支推送到你的 Fork
5. 发起 PR：点击 `Compare & pull request`，填写标题与说明，提交到 `BetterZflyee/deep_agent`

## 分支与提交约定
- 分支命名：`feature/...`、`fix/...`、`docs/...`
- 提交信息：简短描述改动，例如 `feat: 添加市场Agent示例`、`fix: 修复README链接`
- 一个 PR 尽量只做一类改动，便于评审

## PR 要求
- 说明改动目的、主要变更点与影响范围
- 如有截图或示例，请附上
- 若涉及文档，请同步更新 `README.md` 或相应说明文件

## 评审与合并
- 维护者会在 `Pull Requests` 页面评审你的改动
- 通过后会使用 `Squash and merge` 合并；主分支启用了保护，需要通过 PR 才能合并

## 注意事项
- 不要提交任何密钥或隐私数据
- 大文件或二进制不建议直接入库
- 如不确定规范，可先发起草稿 PR 进行讨论
